## WHY THIS MATTERS

The messaging matrix is the connective tissue between research and execution. Without it, every agent writes copy from scratch — the ad writer guesses at pain points, the LP writer invents proof points, the email writer picks random angles. You get Frankenstein messaging: technically competent copy that doesn't compound because nothing connects to anything else.

With it, every piece of copy across every channel pulls from the same validated positioning. The Researcher's ICP insights flow into the Content Writer's headlines. The CRO agent's objection data feeds the email sequences. The Paid agent knows which angles to test first because the matrix ranks them.

Fill this once. Update it when you learn something. Never write copy without checking it first.

## 1. CORE POSITIONING

```
ONE-LINER
─────────
[What you do] for [who you do it for] so they can [outcome that matters].

Example: "We automate invoice reconciliation for mid-market CFOs so they
close the books in days, not weeks."

Rules:
- One sentence. If you need two, you don't understand your product yet.
- Must pass the taxi test: a stranger should get it in 10 seconds.
- No jargon, no superlatives, no "platform" or "solution."
```

```
CATEGORY
────────
Market category:   [The category you compete in — name it plainly]
Category context:  [Is the category established, emerging, or one you're creating?]

Rules:
- If you're creating a category, state the adjacent known category
  and how you diverge. ("Like [known thing] but [key difference].")
- Never claim a category you can't defend in a competitive conversation.
```

```
DIFFERENTIATION
───────────────
Differentiator 1:  [The thing that is structurally different — not "better," different]
Why it matters:    [What this difference enables for the customer]

Differentiator 2:  [Second structural difference, if it exists]
Why it matters:    [What this difference enables for the customer]

Rules:
- "Better UX" is not a differentiator. "Only product that does X" is.
- "Faster" is not a differentiator unless you can quantify the gap (10x, not 2x).
- If you can't name two genuine differentiators, name one and own it completely.
- Test: would a competitor agree this is different (even if they'd argue it doesn't matter)?
```

## 3. THE MATRIX

This is the core artifact. One row per ICP segment × pain point combination. Most clients have 2–3 ICPs × 3–5 pain points = 6–15 rows. This is the source of truth for all copy.

```
MESSAGING MATRIX
────────────────────────────────────────────────────────────────────────────────

| ICP Segment | Pain Point | Value Prop | Proof Point | Headline Angle | CTA |
|-------------|------------|------------|-------------|----------------|-----|
| [Segment 1] | [Pain 1 — use their words, not yours] | [Which value prop addresses this] | [Specific evidence: number, case study, mechanism] | [Draft headline that leads with this angle] | [Specific action + outcome] |
| [Segment 1] | [Pain 2] | [Value prop] | [Proof] | [Headline angle] | [CTA] |
| [Segment 1] | [Pain 3] | [Value prop] | [Proof] | [Headline angle] | [CTA] |
| [Segment 2] | [Pain 1] | [Value prop] | [Proof] | [Headline angle] | [CTA] |
| [Segment 2] | [Pain 2] | [Value prop] | [Proof] | [Headline angle] | [CTA] |
| [Segment 2] | [Pain 3] | [Value prop] | [Proof] | [Headline angle] | [CTA] |
| [Segment 3] | [Pain 1] | [Value prop] | [Proof] | [Headline angle] | [CTA] |
| [Segment 3] | [Pain 2] | [Value prop] | [Proof] | [Headline angle] | [CTA] |
| [Segment 3] | [Pain 3] | [Value prop] | [Proof] | [Headline angle] | [CTA] |

────────────────────────────────────────────────────────────────────────────────
```

**How to fill the matrix:**

1. Start with ICP segments from the Researcher's ICP profiles. Use the exact segment names.
2. Pull pain points from ICP research — verbatim customer language beats your paraphrase every time.
3. Map each pain to the value prop that directly addresses it. One pain can map to one value prop. If it maps to three, you haven't been specific enough.
4. Proof points must be concrete. "Customers love us" is not proof. "Reduced churn by 34% for [company]" is proof.
5. Headline angles are drafts, not final copy. They set the direction. The Content Writer refines from here.
6. CTAs must be specific. Never "Learn More." Always "[Action verb] + [specific outcome]."

**Rules:**
- Every row must have all six columns filled. Gaps mean you're guessing.
- If you don't have a proof point for a row, flag it — that's a research gap, not a copy problem.
- Update when: new ICP data arrives, a message test wins/loses, a new case study lands.
- The Paid agent, Content Writer, CRM agent, and CRO agent all reference this matrix. Keep it current.

**Rules:**

- Rank objections by frequency, not severity. The one that comes up in 80% of calls gets handled first.

## 7. MESSAGE TESTING PRIORITIES

Not all messages are equal. Test the ones with the highest uncertainty and highest potential impact first.

```
TESTING PRIORITY MATRIX
────────────────────────

| Priority | Message / Angle | Uncertainty | Impact | Test Method | Channel | Status |
|----------|----------------|-------------|--------|-------------|---------|--------|
| 1 | [Message with highest uncertainty × impact] | [H/M/L] | [H/M/L] | [A/B ad copy, LP headline test, email subject test] | [Channel] | [Not started / Running / Complete] |
| 2 | [Next priority] | [H/M/L] | [H/M/L] | [Test method] | [Channel] | [Status] |
| 3 | [Next priority] | [H/M/L] | [H/M/L] | [Test method] | [Channel] | [Status] |
| 4 | [Next priority] | [H/M/L] | [H/M/L] | [Test method] | [Channel] | [Status] |
| 5 | [Next priority] | [H/M/L] | [H/M/L] | [Test method] | [Channel] | [Status] |
```

**How to structure a messaging test:**

- **A/B ad copy test:** Run 2–3 ad variants with different headline angles from the matrix. Same audience, same creative, same CTA. Isolate the message. Minimum 1,000 impressions per variant before reading results.
- **LP headline test:** Swap the hero headline only. Keep everything else identical. Measure bounce rate and conversion rate. Run for 200+ conversions per variant.
- **Email subject line test:** Test 2–3 subject lines from different angles. Same body copy. Measure open rate and click rate. Minimum 1,000 sends per variant.
- **Positioning test (longer cycle):** Run two different LP versions against the same paid traffic for 2–4 weeks. Measures not just clicks but downstream metrics (MQL, pipeline, close rate).

**Rules:**
- Never test two things at once. One variable per test.
- Always have a hypothesis before testing: "We believe [angle] will outperform [current] because [reason]."
- Log results back into this matrix. Update the "Headline Angle" column with winning messages.
- Kill losing messages. Don't keep running copy that lost a fair test because someone internally likes it.
